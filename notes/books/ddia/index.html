  • What does ACID stand for in transactions? ACID in the transactions stand for A : Atomicity C : Consistency I : Isolation D : Durability ADID transactions provided by the database systems provide the safety guarantees, so that the application can be build with needing to worry about these properties.

  • What is atomicity? Atomicity in transactions means that writes to the database will happen in all-or-nothing i.e. if there is an in middle of the transaction, then the partial results should be discarded.

  • How concurrency atomicity different from database attomicity? Atomicity in thread concurrency is discussed as a thread not able to access partial data while another thread is executing atomic operation. This is achieved using locks or using processor levels atomic operations.

    In terms of databases, atomicity is not about concurrency. Atomicity doesn’t talk about accessing partial data access by concurrent connections as that is covered using Isolation rather it is about when a fault occurs which doing several writes in a transaction, transaction should disacrd or undo all the partial writes until that point and error out. According to the book a better word for Atomicity should be Abortability rather than Atomicity.
